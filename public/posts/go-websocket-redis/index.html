Go Websocket Redis

December 26, 2024

Introduction#

Real-time apps are everywhere—chat systems, live notifications, and dashboards all rely on instant updates. In this blog, we’ll show you how to build a simple subscribe service using Go, WebSocket, and Redis. WebSockets will let the server send real-time updates to clients, while Redis will act as a message broker to manage and send updates.

By the end of this guide, you’ll have a working service where users can receive updates in real-time. We’ll walk through:

  • Setting up WebSocket connections in Go.
  • Using Redis Pub/Sub to manage messages.
  • Connecting everything into one real-time system.

Let’s get started and build something useful!

Let’s get started#

Setting up the project#

To start building our subscribe service, let’s set up a Go project. Begin by creating a new directory for the project and initializing it with Go modules. Open your terminal and run:

mkdir go-ws-redis
cd go-ws-redis
go mod init go-ws-redis

This will create a go.mod file to manage the project’s dependencies. Next, we’ll write a simple HTTP server to ensure everything is working correctly. Create a file called main.go and add the following code:

package main

import (
	"log"
	"net/http"
)

func main() {
	http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
		w.Write([]byte("Hello, World!"))
	})

	log.Println("Server running on :8080")
	log.Fatal(http.ListenAndServe(":8080", nil))
}

Run the server with go run main.go, then visit http://localhost:8080 in your browser. You should see “Hello, World!” displayed on the page. With this, your Go project is set up and ready for the next steps.

Add the WebSocket connection#

Websocket is a protocol that allows for full-duplex communication between a client and a server. It’s ideal for use cases like real-time notifications, chat applications, or our subscribe service.

Before implementing the WebSocket server, let’s add a simple UI to test the WebSocket connection. In the main.go file, add the following code to serve a simple HTML page:
